# `design/_example/` — bundled reference

This folder is **shipped with the template**, not auto-generated by `/design`. Its purpose is to show the complete directory shape that `/design` produces, so you (and Claude) have a concrete reference when generating or reviewing real designs.

## What's inside

```
_example/
├── README.md         ← this file
├── latest.html       ← HTML meta-refresh → v1/preview.html
├── LATEST.txt        ← machine-readable: "v1"
├── LATEST.md         ← markdown link to latest spec
├── CHANGELOG.md      ← Keep-a-Changelog format
└── v1/
    ├── preview.html  ← self-contained Tailwind CDN mockup
    └── DESIGN.md     ← full design spec
```

## Try it

Open `latest.html` in any browser (double-click on Windows, `open latest.html` on macOS). You'll land on the Dashboard preview with:

- Light / dark theme toggle (top-right)
- State switcher (Ready / Loading / Empty / Error)
- Viewport badge (bottom-right) — resize the window to see responsive breakpoints
- No build, no dev server, no dependencies

## Safe to delete

Once you've run `/design` at least once and understand the output shape, you can delete this folder. It's just a reference — nothing in the codebase depends on it.

## When `/design` generates *your* designs

- They go to `design/<your-slug>/v1/`, `v2/`, etc.
- Screenshots (`preview-*.png`) are auto-gitignored — only `preview.html` + `DESIGN.md` + pointers are tracked
- `_example/` screenshots (if you add any) are **exempt** from gitignore — they stay in the repo as reference
